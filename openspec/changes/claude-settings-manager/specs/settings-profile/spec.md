## ADDED Requirements

### Requirement: 创建配置档案
系统 SHALL 允许用户创建新的配置档案，提供档案名称和初始 settings.json 内容。档案名称 MUST 在所有档案中唯一。

#### Scenario: 通过空模板创建档案
- **WHEN** 用户点击"新建档案"按钮并输入名称（如 "OpenAI"）
- **THEN** 系统创建一个以空 `{}` 为初始内容的新档案，名称为 "OpenAI"

#### Scenario: 通过导入文件创建档案
- **WHEN** 用户点击"新建档案"并选择导入一个本地 JSON 文件
- **THEN** 系统读取该文件内容作为档案的 settings.json，以用户指定的名称保存

#### Scenario: 名称重复时拒绝创建
- **WHEN** 用户尝试创建一个与已有档案同名的配置档案
- **THEN** 系统显示错误提示"档案名称已存在"，不创建档案

### Requirement: 列出所有配置档案
系统 SHALL 在主界面显示所有已创建的配置档案列表，包含档案名称、创建时间和是否为活跃状态。

#### Scenario: 显示档案列表
- **WHEN** 用户打开应用主界面
- **THEN** 系统显示所有配置档案的列表，每个条目显示名称、创建/更新时间和活跃标记

#### Scenario: 无档案时的空状态
- **WHEN** 用户打开应用但尚未创建任何配置档案
- **THEN** 系统显示空状态引导提示，建议用户创建第一个档案

### Requirement: 切换活跃配置档案
系统 SHALL 允许用户选择一个档案作为活跃档案。切换活跃档案时 MUST 将该档案的内容写入 Claude Code 的 `settings.json` 文件（覆盖）。同一时间仅允许一个活跃档案。

#### Scenario: 切换到目标档案
- **WHEN** 用户选择档案 "Anthropic" 并点击"切换"按钮
- **THEN** 系统将 "Anthropic" 档案的内容写入 Claude Code 的 settings.json，标记该档案为活跃状态，其他档案标记为非活跃

#### Scenario: 切换失败时回滚
- **WHEN** 用户切换活跃档案但写入 settings.json 失败（如权限不足）
- **THEN** 系统保持原活跃档案不变，显示错误信息告知用户失败原因

### Requirement: 重命名配置档案
系统 SHALL 允许用户修改已有档案的名称，新名称 MUST 在所有档案中唯一。

#### Scenario: 成功重命名
- **WHEN** 用户将档案 "test-config" 重命名为 "OpenAI-Production"
- **THEN** 系统更新档案名称为 "OpenAI-Production"，档案内容不变

#### Scenario: 重命名为已存在名称
- **WHEN** 用户尝试将档案重命名为一个已存在的档案名称
- **THEN** 系统拒绝操作并提示"档案名称已存在"

### Requirement: 复制配置档案
系统 SHALL 允许用户复制已有档案创建副本。副本 MUST 拥有独立的名称和内容，对副本的修改不影响原始档案。

#### Scenario: 复制档案创建副本
- **WHEN** 用户选择档案 "OpenAI" 并点击"复制"按钮
- **THEN** 系统创建名为 "OpenAI (副本)" 的新档案，内容与原档案完全相同

#### Scenario: 多次复制同一档案
- **WHEN** 用户连续复制 "OpenAI" 档案两次
- **THEN** 系统依次创建 "OpenAI (副本)" 和 "OpenAI (副本 2)"

### Requirement: 删除配置档案
系统 SHALL 允许用户删除已有档案。活跃档案 MUST 在删除前提示用户确认。

#### Scenario: 删除非活跃档案
- **WHEN** 用户选择一个非活跃档案并点击"删除"
- **THEN** 系统弹出确认对话框，确认后删除该档案

#### Scenario: 删除活跃档案需额外确认
- **WHEN** 用户尝试删除当前活跃档案
- **THEN** 系统弹出警告对话框，提示"这是当前活跃的配置档案，删除后将恢复为默认配置"，用户确认后删除并清除活跃状态

### Requirement: 显示当前活跃档案
系统 SHALL 在主界面显著位置标识当前活跃的配置档案。

#### Scenario: 高亮活跃档案
- **WHEN** 用户查看档案列表
- **THEN** 当前活跃档案显示醒目的"活跃"标记（如高亮边框或图标），其他档案无此标记

#### Scenario: 无活跃档案时的提示
- **WHEN** 用户尚未切换任何档案为活跃状态
- **THEN** 列表中无"活跃"标记，主界面提示"尚未激活任何配置档案"

### Requirement: 系统托盘快速切换
系统 SHALL 在系统托盘提供右键菜单，列出所有配置档案供快速切换。

#### Scenario: 托盘菜单列出档案
- **WHEN** 用户右键点击系统托盘图标
- **THEN** 菜单中列出所有配置档案，当前活跃档案旁显示勾选标记

#### Scenario: 从托盘切换档案
- **WHEN** 用户在托盘菜单中选择一个非活跃档案
- **THEN** 系统将该档案切换为活跃状态，写入 settings.json，无需打开主窗口
