## ADDED Requirements

### Requirement: 添加远程主机
系统 SHALL 允许用户添加远程主机连接信息，包含主机名称、地址、端口、用户名和认证方式（密码或密钥）。

#### Scenario: 通过密码认证添加主机
- **WHEN** 用户填写主机名称 "dev-server"、地址 "192.168.1.10"、端口 "22"、用户名 "admin"，选择密码认证并输入密码
- **THEN** 系统将主机信息保存到数据库，密码 MUST 使用 AES-256-GCM 加密后存储

#### Scenario: 通过 SSH 密钥认证添加主机
- **WHEN** 用户选择密钥认证并指定本地 SSH 私钥文件路径
- **THEN** 系统保存主机信息，记录密钥路径而非密码

#### Scenario: 验证主机连通性
- **WHEN** 用户在添加主机时点击"测试连接"按钮
- **THEN** 系统尝试使用提供的凭据建立 SSH 连接，成功则提示"连接成功"，失败则显示具体错误

### Requirement: 列出所有远程主机
系统 SHALL 在远程主机管理面板中显示所有已配置的主机列表。

#### Scenario: 显示主机列表
- **WHEN** 用户打开远程主机管理面板
- **THEN** 系统显示所有已配置主机的列表，每条显示名称、地址、端口和最后同步时间

#### Scenario: 无主机时的空状态
- **WHEN** 用户尚未添加任何远程主机
- **THEN** 系统显示空状态提示"尚未添加远程主机"，引导用户添加第一台主机

### Requirement: 编辑远程主机信息
系统 SHALL 允许用户修改已有主机的连接信息。

#### Scenario: 修改主机地址
- **WHEN** 用户将主机 "dev-server" 的地址从 "192.168.1.10" 修改为 "10.0.0.5"
- **THEN** 系统更新该主机的地址信息

#### Scenario: 更新认证密码
- **WHEN** 用户修改某主机的登录密码
- **THEN** 系统使用新密码重新加密并存储，旧密码 MUST 被彻底覆盖

### Requirement: 删除远程主机
系统 SHALL 允许用户删除已配置的远程主机。

#### Scenario: 删除主机
- **WHEN** 用户选择主机 "dev-server" 并点击"删除"
- **THEN** 系统弹出确认对话框，确认后删除该主机及其所有同步历史记录

### Requirement: 凭据安全存储
系统 MUST 确保所有密码凭据以加密形式存储，绝不允许明文存储密码。

#### Scenario: 密码加密存储
- **WHEN** 用户保存包含密码的主机连接信息
- **THEN** 系统使用 AES-256-GCM 算法加密密码，密钥由操作系统密钥管理器（macOS Keychain / Linux Secret Service / Windows Credential Manager）提供

#### Scenario: 密码读取解密
- **WHEN** 系统需要使用已存储的密码进行 SSH 连接
- **THEN** 系统从操作系统密钥管理器获取加密密钥，解密密码用于连接，解密后的密码仅在内存中短暂存在，不写入日志或磁盘

#### Scenario: 操作系统密钥管理器不可用时的回退
- **WHEN** 操作系统密钥管理器不可用（如无桌面环境的 Linux）
- **THEN** 系统使用应用级主密钥文件加密存储，首次使用时要求用户设置主密码
